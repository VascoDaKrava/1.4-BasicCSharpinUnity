5.1 Расписать в текстовом документе плюсы и минусы изученных структур данных и расписать примеры в которых нужно будет использовать тот или иной контейнер

1. List
Плюсы - Быстрая итерация.
Минусы - Затратно O(n) добавлять/удалять новые элементы при большом списке. При удалении элемента, все остальные до конца списка сдвигаются, чтобы "занять" опуствешее место. Т.о. происходит перезапись ссылок в количестве от позиции удалённого элемента и до конца списка.
Использование - Простой список для хранения/добавления/удаления данных. Используется когда не нужны дополнительные функции, а элементы могут удаляться в произвольном месте.

2. HashSet / SortedSet
Плюсы - Сложность манипуляций с данными постоянна. Быстрый поиск по значению.
Минусы - Медленная итерация.
Использование - Хранение заведомо уникальных элементов. Например, коллайдеров.

3. ObservableCollection
Плюсы - Аналогично List.
Минусы - Аналогично List.
Использование - Когда необходимо уведомлять об изменениях в коллекции.

4. LinkedList
Плюсы - Быстрое добавление/удаление элементов.
Минусы - Долгий поиск. При поиске проходится каждый элемент до указанной позиции.
Использование - Список с возможностью вставки элемента в позицию. Используется когда необходим функционал обычного List, но важен порядок элементов. Например, для хранения списка действий, которые должны идти в определённой последовательности с возможностью добавления или удаления элементов/действий по конкретной позиции, а также, когда планируется частое удаление/добавление элементов в произвольных местах и не нужен частый поиск.

5. Dictionary
Плюсы - Сложность манипуляций с данными постоянна. Быстрая итерация. Быстрый поиск по значению.
Минусы - Относительно большой объём служебных данных.
Использование - Например, для частотного массива.

6. Queue
Плюсы - Сложность манипуляций с данными постоянна, т.к. добавление и удаение происходит всегда с крайним элементом. Быстрая итерация.
Минусы - Скорее особенность - работа только с карйним элементом.
Использование - Для хранения данных, которые должны обрабатываться по очереди. Например, при следовании по чек-поинтам маршрута.

7. Stack
Плюсы - Сложность манипуляций с данными постоянна, т.к. добавление и удаение происходит всегда с крайним элементом. Быстрая итерация.
Минусы - Скорее особенность - работа только с карйним элементом.
Использование - Для реализации функционала "Отмена последнего действия".
